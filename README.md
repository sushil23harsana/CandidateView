**For Hiring Managers/Founders**
- Post jobs, receive applications, get AI-ranked candidates with explanations

**For Candidates**
- Easy application process, auto-filled forms via resume parsing, no Workday nightmare

```python
import json
from langchain.chat_models import ChatOpenAI
from langchain.prompts import PromptTemplate
import pdfplumber
from docx import Document

class ResumeParser:
    def __init__(self):
        self.llm = ChatOpenAI(model="gpt-4", temperature=0)
    
    def extract_text(self, file_path: str) -> str:
        """Extract raw text from PDF or DOCX"""
        if file_path.endswith('.pdf'):
            return self._extract_pdf(file_path)
        elif file_path.endswith('.docx'):
            return self._extract_docx(file_path)
        else:
            raise ValueError("Unsupported file format")
    
    def _extract_pdf(self, file_path: str) -> str:
        text = ""
        with pdfplumber.open(file_path) as pdf:
            for page in pdf.pages:
                text += page.extract_text() or ""
        return text
    
    def _extract_docx(self, file_path: str) -> str:
        doc = Document(file_path)
        return "\n".join([para.text for para in doc.paragraphs])
    
    def parse(self, file_path: str) -> dict:
        """Main parsing function"""
        # Step 1: Extract raw text
        raw_text = self.extract_text(file_path)
        
        # Step 2: Use LLM to structure it
        prompt = PromptTemplate(
            template=RESUME_PARSER_PROMPT,
            input_variables=["resume_text"]
        )
        
        response = self.llm.predict(
            prompt.format(resume_text=raw_text)
        )
        
        # Step 3: Parse JSON response
        try:
            parsed_data = json.loads(response)
            return {
                "success": True,
                "data": parsed_data,
                "raw_text": raw_text
            }
        except json.JSONDecodeError:
            return {
                "success": False,
                "error": "Failed to parse resume",
                "raw_text": raw_text
            }
```

### Scoring Engine Implementation (Current)
- **Configurable per job:** Category weights and resume sub-weights are stored per job and normalized to 100%. Each job can override defaults without changing code.  
- **Inputs:** Resume analysis (skills match + experience fit) and GitHub analysis (code quality, documentation, engineering practices, project originality).  
- **Normalization:** Category scores are normalized to a 0â€“100 scale. If a category is missing (e.g., no GitHub), the total score is re-weighted across available categories while keeping missing categories in the breakdown with an explanation.  
- **Project originality detection:** Uses GitHub project signals (tutorial indicators, green flags) to compute an average originality score and explanation.  
- **Human-readable explanations:** Each category produces a short explanation, and the final output concatenates them into a candidate scoring summary.  
- **Persistence:** The scoring job saves the total score, per-category breakdown, and explanation text in a scoring results store.  
- **Background job triggers:** Scoring runs once resume parsing completes and GitHub analysis finishes (or GitHub is missing and not required). The scoring engine is not exposed via API yet.

```python
def detect_tutorial_project(repo_analysis: dict) -> dict:
    """
    Detect if a project is original work or tutorial copy.
    Important: We DON'T penalize for low commit frequency!
    """
    
    red_flags = []
    green_flags = []
    
    # Check README content
    readme = repo_analysis.get("readme_content", "").lower()
    
    # Red flags (indicates tutorial)
    tutorial_keywords = [
        "tutorial", "course", "udemy", "coursera", "freecodecamp",
        "follow along", "learning", "practice", "exercise",
        "clone of", "copy of", "based on tutorial"
    ]
    
    for keyword in tutorial_keywords:
        if keyword in readme:
            red_flags.append(f"README contains '{keyword}'")
    
    # Generic project names
    generic_names = [
        "todo", "calculator", "weather-app", "crud", "blog",
        "e-commerce", "chat-app", "social-media-clone"
    ]
    
    repo_name = repo_analysis.get("name", "").lower()
    for name in generic_names:
        if name in repo_name:
            red_flags.append(f"Generic project name: {repo_name}")
    
    # Green flags (indicates original work)
    # Custom branding/name
    if not any(name in repo_name for name in generic_names):
        green_flags.append("Unique project name")
    
    # Problem statement in README
    problem_keywords = ["problem", "solves", "built this because", "needed"]
    if any(keyword in readme for keyword in problem_keywords):
        green_flags.append("Has problem statement")
    
    # Features section
    if "features" in readme or "what it does" in readme:
        green_flags.append("Has features description")
    
    # Live demo
    if "demo" in readme or "live" in readme:
        green_flags.append("Has live demo")
    
    # Calculate originality score
    originality_score = 50  # Start neutral
    originality_score -= len(red_flags) * 15
    originality_score += len(green_flags) * 10
    originality_score = max(0, min(100, originality_score))
    
    return {
        "is_likely_tutorial": len(red_flags) > len(green_flags),
        "originality_score": originality_score,
        "red_flags": red_flags,
        "green_flags": green_flags
    }
```

### Recommended Stack (Open Source + Free)
- **Backend:** FastAPI + PostgreSQL + SQLModel/SQLAlchemy
- **Frontend:** Next.js + TailwindCSS + shadcn/ui
- **Auth:** Auth.js (NextAuth) with Google/GitHub OAuth
- **Async jobs:** Celery + Redis

### Why FastAPI for Backend
- **API-first**: Fits the two-sided platform and dashboard use cases.
- **Async-ready**: Ideal for resume parsing, GitHub analysis, and AI scoring jobs.
- **Modern tooling**: Pydantic v2, OpenAPI docs, dependency injection.

### Why Next.js for Frontend
- **Dashboard-friendly**: Great UX for hiring manager and candidate portals.
- **Open-source ecosystem**: TailwindCSS + shadcn/ui + Radix for accessible UI.
- **Developer velocity**: Fast iteration with App Router and React Query.

### Auth Strategy
- **Start with Auth.js (NextAuth)**: Open-source, easy to self-host, supports email and OAuth.
- **Scale with Keycloak** if SSO/RBAC is required later.
